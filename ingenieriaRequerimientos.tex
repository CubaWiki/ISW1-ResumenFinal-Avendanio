\chapter{Ingeniería de Requerimientos}

\section{Introducción}

\subsection{Definición}

Todo Software se desarrolla con un \emph{propósito}. Luego podemos definir \emph{calidad} como el grado con que el software cumple con el propósito.

La \emph{\textbf{Ingeniería de Requerimientos (RE)}} es un conjunto de actividades concernientes a identificar y comunicar el propósito de un sistema intensivo de software y el contexto en que será usado. 

Tanto el problema como la solución son el resultado del comportamiento emergente de la interacción entre los componentes del sistema. Por eso hablamos de Ingeniería de Requerimientos de \emph{Sistemas Intensivos en Software} (Software + Hardware + Entorno).

Por lo tanto la Ingeniería de Requerimientos actúa como puente entre las necesidades del mundo real de los usuarios, clientes y otros individuos afectados por el sistema de software, y las capacidades y oportunidades surgidas a partir de las tecnologías de software intensivo.

\subsection{Separación del problema y la solución}

\includegraphics[width = 0.9\textwidth]{Graficos/separacion.png}

La Ingeniería de Requerimientos requiere controlar que:

\begin{itemize}
	\item El problema se corresponde con lar verdaderas necesidades (validación)
	\item La solución correctamente resuelve la descripción del problema (verificación)
\end{itemize}

\subsection{Actividades de Ingeniería de Requerimientos}

\begin{description}
	\item[Elicitación:] evocar una respuesta de alguien como reacción a preguntas o acciones.
	\item[Modelado:] documentar rigurosamente para análisis (abstraer y estructurar lo elicitado)
	\item[Análisis:] verificación inter e intra modelos
	\item[Validación:] ¿los modelos se corresponden con la realidad?
	\item[Priorización:] establecer criterios de preferencia con respecto a alcance y objetivos.
	\item[Negociación:] unificar criterios entre los interesados
	\item[Especificación:] documentación completa y detallada (documento final) 
\end{description}
